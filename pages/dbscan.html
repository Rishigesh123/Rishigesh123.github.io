DBSCAN

Density-Based Spatial Clustering of Applications with Noise.

DENSITY

The Intuition

DBSCAN is a density-based clustering algorithm. Unlike K-Means, it doesn't assume clusters are round. It groups together points that are close to each other in high-density areas.

Imagine finding cities on a map. You look for areas where houses are packed tightly together. The empty space between cities is "noise."

The Core Parameters

  • Epsilon (eps): The maximum distance between two points for them to be considered neighbors.
  • MinPoints: The minimum number of neighbors a point must have to be considered a "Core Point."

Point Classification

  1. Core Points: Points that have at least MinPoints within eps distance.
  2. Border Points: Points within eps distance of a core point but don't have enough neighbors themselves.
  3. Noise Points: Any point that isn't a core point or a border point. These are treated as outliers.

How It Works

The algorithm starts with an unvisited point. If it's a Core Point, it "expands" the cluster by adding all reachable neighbors. It continues until all points in the dense region are found.

ALGORITHM TYPE

Unsupervised (Density Clustering)

BEST USED FOR

Identifying outliers, non-spherical clusters, Geospatial data

STRENGTHS

  • Finds clusters of any shape
  • Robust to outliers (Noise)
  • No need to specify K

WEAKNESSES

  • Struggles with varying densities
  • Hard to choose Epsilon
  • Sensitive to distance metrics
