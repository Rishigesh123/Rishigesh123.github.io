Decision Tree

Mapping complex decisions into simple, logical branches.

LOGIC

The Intuition

A Decision Tree is a supervised learning algorithm that works like a flow chart. It recursively splits data into subsets based on the most important features until it reaches a final prediction.

Think of the game "20 Questions." The algorithm asks the best possible question at each step to narrow down the answer as quickly as possible.

Anatomy of a Tree

  • Root Node: The very first question that represents the entire dataset.
  • Internal Nodes: Questions about specific features (e.g., "Is wind speed > 20mph?").
  • Branches: The outcome of a question (e.g., Yes / No).
  • Leaf Nodes: The final classification or numerical prediction.

How It Splites

The algorithm uses mathematical criteria to decide which feature to split on:

Gini Impurity = 1 - Σ (pᵢ)²

Or Entropy. The goal is to maximize "Information Gain" at every split—making the resulting groups as "pure" as possible.

Overfitting

A major challenge is that trees can become too complex, memorizing the training data instead of learning patterns. We solve this by "Pruning"—cutting off branches that don't add significant predictive power.

ALGORITHM TYPE

Supervised (Classification & Regression)

STRENGTHS

  • Visual & Intuitive
  • Handles non-linear data
  • No data scaling needed

WEAKNESSES

  • Prone to Overfitting
  • Unstable (Small data shifts = big tree shifts)
  • Biased if classes are unbalanced
