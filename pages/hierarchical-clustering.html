Hierarchical Clustering

Building nested groups in a structured, tree-like hierarchy.

HIERARCHY

The Intuition

Hierarchical Clustering is an unsupervised algorithm that groups similar objects into clusters that are themselves grouped into larger clusters.

Think of an ancestry tree. You start with individual family members, group them into nuclear families, then extended families, and so on until the entire tree is connected.

Types of Hierarchy

  • Agglomerative (Bottom-Up): Start with each point as its own cluster and merge the closest ones repeatedly. This is the most common method.
  • Divisive (Top-Down): Start with all points in one giant cluster and split them until each point is solo.

The Dendrogram

The visual output of this algorithm is called a Dendrogram. Itâ€™s a tree-like diagram that shows exactly how and when each merge happened. You can "cut" the tree at any level to choose your number of clusters.

Linkage Criteria

How do we define "closeness"?

  • Single Linkage: Distance between the closest members of two clusters.
  • Complete Linkage: Distance between the farthest members.
  • Ward's Method: Minimizes the variance within each cluster (most robust).

ALGORITHM TYPE

Unsupervised (Clustering)

KEY OUTPUT

Dendrogram (Visual Hierarchy)

STRENGTHS

  • No need to pre-define K
  • Easy to visualize
  • Reveals nested relationships

WEAKNESSES

  • Slow on large datasets
  • Merge decisions are permanent
  • Sensitive to outliers
