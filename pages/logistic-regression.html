Logistic Regression

Decoding choices with the power of probability.

CLASSIFY

The Intuition

Despite its name, Logistic Regression is used for Classification, not regression. It's a supervised learning algorithm that predicts the probability of a binary outcome (Yes/No, Spam/Not Spam).

Imagine a light switch. Instead of being either ON or OFF, logistic regression calculates the chance it's ON based on various sliders (features).

The Sigmoid Function

The "magic" happens using the Sigmoid (or Logistic) function, which squashes any numerical input into a value between 0 and 1.

σ(z) = 1 / (1 + e⁻ᶻ)

If the result is > 0.5, we classify it as 1. If it's < 0.5, it's 0.

Decision Boundary

The algorithm creates a "boundary" that separates classes. For example, in credit approval, it might draw a line where anyone with a credit score above 700 is "Approved."

Log Loss

Instead of MSE, we use Log Loss (Cross-Entropy) to measure how close our predicted probability is to the true result.

Loss = -Σ [yᵢ log(pᵢ) + (1-yᵢ) log(1-pᵢ)]

ALGORITHM TYPE

Supervised (Classification)

BEST USED FOR

Spam detection, Medical Diagnosis (Sick/Healthy), Customer Churn

STRENGTHS

  • Outputs Probabilities
  • Very efficient
  • Robust to slight noise

WEAKNESSES

  • Assumes independence
  • Struggles with non-linear data
  • Prone to binary bias
