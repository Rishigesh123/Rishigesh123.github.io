Principal Component Analysis (PCA)

Simplifying complexity by reducing dimensions without losing essence.

REDUCE

The Intuition

PCA is an unsupervised algorithm used for dimensionality reduction. It transforms a large set of variables into a smaller one that still contains most of the information in the original set.

Imagine compressing a 3D shadow onto a 2D wall while still being able to recognize the object. PCA finds the best angle to capture that shadow.

Core Concepts

  • Variance: Measures how spread out data is. PCA tries to preserve maximum variance.
  • Covariance Matrix: A table showing how variables relate to each other.
  • Eigenvectors & Eigenvalues: These define the directions (Principal Components) and their relative importance.

How It Works

  1. Standardize: Scale the data so all features have the same range.
  2. Variance Analysis: Compute the Covariance Matrix.
  3. Extract Components: Find the Eigenvectors (directions) and Eigenvalues (magnitude).
  4. Sort & Select: Pick the top components that explain the most variance.
  5. Transform: Project the data onto the new, smaller feature space.

Mathematical Foundation

The first Principal Component is the unit vector u₁ that maximizes the variance of the projected data:

Maximize: uᵀ Σ u

Where Σ is the Covariance Matrix of the dataset.

ALGORITHM TYPE

Dimensionality Reduction

BEST USED FOR

Image Compression, Data Visualization, Noise Filtering

STRENGTHS

  • Reduces Overfitting
  • Speeds up Learning
  • Removes Correlated features

WEAKNESSES

  • Loss of Interpretability
  • Needs Standard Scaling
  • Captures only linear relations
